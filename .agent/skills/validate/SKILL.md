---
name: validate
description: Use before commands, tools, browser automation, scraping, installs, deploys, scripts, or large edits.
---


# Steps
1. Define objective.
2. Inspect current state.
3. Pick the lowest-risk approach.
4. Identify files/tools affected.
5. Name risks.
6. Define validation method.
7. Define stop condition.

# Stop Conditions
- Missing credentials.
- Destructive or paid action.
- Unknown project structure.
- No validation path.
- User intent unclear in a way that changes outcome.

# Output Format
```md
Objective:
Current state:
Approach:
Files/tools affected:
Risk:
Validation method:
Stop condition:
```
