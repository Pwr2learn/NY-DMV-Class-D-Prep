# Steps
1. Confirm whether the output is a single HTML file or integrated component.
2. Use editorial art direction, asymmetrical grid, texture, and warm architectural palette.
3. Avoid generic SaaS bento grids and neon/tech visuals unless requested.
4. Build responsive fallbacks; disable custom cursor/hover reveals on touch devices.
5. Validate scroll animations, image treatment, layout, and accessibility.

# Stop Conditions
- Required libraries/CDNs are not approved.
- Design constraints conflict with existing project design system.

# Gotchas
- Do not use Inter/Roboto if the requested art direction forbids them.
