---
name: plan-review
description: Use for complex, high-risk, architectural, security, or large refactor tasks before implementation.
---


# Steps
1. Restate objective in one sentence.
2. List constraints and risks.
3. Review plan for overbuild, missing validation, security gaps, and cost waste.
4. Recommend smallest safer plan.
5. Define stop conditions.

# Output Format
```md
Plan rating:
Main risks:
Recommended plan:
Validation:
Stop conditions:
```
