---
name: browser-preflight
description: Use before browser-heavy Antigravity work, UI QA, scraping, or web automation.
---


# Steps
1. Confirm target URL/page.
2. Confirm browser automation is available.
3. Confirm approval for browser actions if needed.
4. Verify page loads.
5. Capture screenshot/artifact when useful.
6. Define fallback if browser tooling fails.

# Stop Conditions
- Browser actions fail twice.
- Login or paid action required.
- User's active work could be disrupted.
- Target or success criteria unclear.
